## EPICGRAM TikTok Skills Skeleton

### 1. `epicgram-tiktok-brief/SKILL.md`

```md
---
name: epicgram-tiktok-brief
description: Turn a raw Telegram or operator idea into a structured TikTok content brief with hook, format, CTA, tone, duration, and audience.
---

# epicgram-tiktok-brief

## Summary
Create a TikTok-first content brief from raw ideas, notes, messages, or campaign goals.

## Inputs
- account_id: string
- project_id: string
- idea_text: string
- source_context: optional string
- campaign_goal: optional string
- target_audience: optional string
- duration_target: optional enum (15s, 30s, 45s, 60s)
- tone: optional string

## Outputs
- brief_title: string
- content_angle: string
- hook_direction: string
- video_format: enum (talking_head, meme, tutorial, reaction, storytime, duet_like)
- cta: string
- target_duration: string
- audience_profile: string
- constraints: array
- risk_flags: array

## Tools
- POST /api/skills/tiktok/brief
- GET /api/accounts/:account_id/identity

## Safety
- Must not publish or schedule anything.
- Should attach risk notes for sensitive topics.
```

### 10. `epicgram-tiktok-safe-mode/SKILL.md`

```md
---
name: epicgram-tiktok-safe-mode
description: Evaluate TikTok actions for policy, abuse, approval, rate limit, and operational risk before publication or automation.
---

# epicgram-tiktok-safe-mode

## Inputs
- action_type: enum (publish, schedule, edit, delete, bulk_publish, bulk_edit)
- account_id: string
- content_payload: object
- operator_role: string
- requested_volume: optional integer

## Outputs
- decision: enum (allow, needs_approval, block)
- risk_score: number
- policy_flags: array
- required_approvals: array
- reasoning: array

## Tools
- POST /api/skills/tiktok/safe-mode
- POST /api/moderation/check
- GET /api/accounts/:account_id/rate-limits

## Safety
- This skill is itself the safety layer and must be called before side effects.
- Bulk actions should default to needs_approval or block unless explicitly allowed.
```
